API Reference

The MathXplains API is a REST API for general operations, giving access to all the features in the plataform.

Base URL
https://api.mathxplains.com.br

Authentication

Authenticating with the MathXplains API is done with JWT tokens. Authentication is performed with the Authorization HTTP header in the format Authorization: Bearer TOKEN.

Most endpoints require a valid authentication. Endpoints that do not enforce any kind of authentication will have a warning.

Date/Time Format

MathXplains utilizes the ISO8601 format for all timestamps returned in our models.

Nullable/Optional Fields

Fields that may return null have types prefixed with a question mark. Optional fields have their names suffixed with a question mark.

When making a request, required fields will always have their names suffixed with a * symbol.

See examples:

Field Type
optional_field? boolean
nullable_field ?string
both? ?string
send_required* integer

Rate Limiting

The API Gateway implements limitations, preventing an excessive amount of requests. You can make an average of 5 requests per second per endpoint. Learn more.

Undocumented endpoints are likely to have different rate limiting rules. Better just not try to use them :)

Content Type

All the data transmitted in the MathXplains API uses the JSON format by default, and requires the Content‐Type: application/json HTTP header to be present.

Boolean Query Parameters

Since query parameters do not directly support boolean values, the MathXplains API uses true and false as boolean representations.

See examples:

✓ ‐‐ https://api.mathxplains.com/subjects?available=true
✕ ‐‐ https://api.mathxplains.com/subjects?available=True
✕ ‐‐ https://api.mathxplains.com/subjects?available=TRUE

Endpoints

These are all the endpoints, giving access to every resource/feature in the plataform.

Professors

Get All Professors

Returns all the existing professors.

This endpoint does not require authentication.
GET /professors

Query Parameters

Name Type Description
known boolean Displays only known professors (default: false).

Response Body

Name Type Description
professors Array of professor objects. All the expected professors.

Appointments

Get All Appointments

Returns all the existing appointments.

GET /appointments

Query Parameters

Name Type Description
status A valid status (case-insensitive). Displays only appointments with the given status.

Response Body

Name Type Description
appointments Array of appointment objects. All your appointments.

Create Appointment

Creates a new appointment. Returns an appointment object on success.

Limitations
  • topic must be in range of 5 and 30 characters (inclusive).
  • description must be in range of 10 and 1000 characters (inclusive).
  • Only available subjects can be provided, otherwise you will get a 400 Bad Request.
  • The scheduled_at field cannot be in the past and more than 1 month in the future.
The MathXplains API may strip certain characters and remove trailing/leading spaces, in these cases, they will not count towards the range constraints.
All users have a limit of 10 active (per the is_active field) appointments. This endpoint will fail with a 400 Bad Request if this constraint is violated.
POST /appointments

Request Body

Name Type Description
topic* string The topic/title of the appointment.
description* string The description of the appointment.
subject_id* integer The ID of the subject of the appointment.
scheduled_at* ISO8601 date When the user must attend to the scheduled appointment.

The scheduled_at field must be in date‐only format: 2024‐01‐18.

Delete Appointment

Deletes an appointment by its ID. Returns a 204 No Content on success. The request will not fail if the resource for the given ID does not exist.

This endpoint does not delete appointments older than 24 hours.
DELETE
/appointments/{appointment.id}

Users

Returns a user object.

This endpoint is admin‐only (unless providing your own id).
GET /users/{user.id}

Get Self User

Returns the self user.

GET /users/@me

Get Users

Returns all the existing users.

This endpoint is admin‐only.
GET /users

Response Body

Name Type Description
users Array of user object. All the existing users.

Create User

Creates a new user with the given credentials. Returns a 204 No Content on success.

Limitations
  • name must be in range of 3 and 64 characters (inclusive).
  • email must be in range of 6 and 128 characters (inclusive). Regex: [a-z0-9._%+\-]+@[a-z0-9.\-]+\.[a-z]{2,}$.
  • password must be in range of 8 and 128 characters (inclusive), have at least one lower and uppercase character and one number.
This endpoint does trigger an email confirmation to the recipient's email.
This endpoint does not require authentication.
POST /users

Request Body

Name Type Description
name* string The username of the account.
email* string The email of the account.
password* string The password of the account.

Create Email Confirmation

Verifies the email address of the account being created. Returns a 204 No Content on success.

This endpoint does not require authentication.
POST /users/verify

Request Body

Name Type Description
email* string The email to be confirmed.
code* string The code sent in the recipient's email address.

Sign In User

Signs In the user with the given credentials.

This endpoint does not require authentication.
POST /users/login

Request Body

Name Type Description
email* string The email of the user to sign in.
password* string The password of the user to sign in.

Response Body

Name Type Description
token string The JWT token used to authorize the requests.
refresh_token string The refresh token, used to generate a new token.

Entities

All MathXplains endpoints return a pre-defined model/entity. These structures are defined and explained (if needed) below.

Professors

Name Type Description
id integer The ID uniquely representing the professor.
name string The "common" name of the professor.
full_name string The full name of the professor.
common_name * ?string How people usually call the professor.
known ** boolean Whether it's a known professor or not.

* Represents how people usually call the given professor, if this field is null, then it means people already call the professor by their name, your application should fall back to name in this case.

** This field is true when the given professor has conducted classes for the 3A class in the past (or still does), false otherwise.

Subjects

Name Type Description
id integer The ID uniquely representing the subject.
professor_id * ?integer The ID of the professor responsible for this subject.
name string The "common" name of the subject.
full_name string The full/real name of this subject.
available ** boolean Whether it is available for usage or not.

* This field will be null if the subject does exist and is available, but no professors currently conduct it for the 3A class.

** MathXplains API restricts the usage of some subjects due to the real intentions of its creation, in general, only subjects related to mathematical operations are available and accepted by the API when invoking endpoints like Create Appointment.

Appointments

Name Type Description
id integer The ID uniquely representing the appointment.
topic string The topic of the appointment.
description string The description of this appointment.
user_id string The ID of the user who created this appointment.
subject_id integer The ID of the subject of this appointment.
professor_id ?integer The ID of the professor responsible for the subject used in this appointment.
scheduled_at ISO8601 date When the appointment is scheduled to.
rejected * boolean Whether this appointment was rejected by an admin or not.
is_active boolean Whether this appointemnt is still active or not.
created_at ISO8601 timestamp When the returned appointment was created.
updated_at ISO8601 timestamp The last time the appointment was updated.

* Appointments can be rejected by several reasons, such as: undescriptive topic/description, subject and topic not matching, excessive amount of questions etc.

User

Name Type Description
id string The ID uniquely for this user.
name string The name the user has registered.
is_admin boolean Whether the user has admin privileges or not.
email_verified boolean Whether the user has verified their email or not.
verified_at ?ISO8601 When the user verified their email.
created_at ISO8601 timestamp When the account was successfully created.
updated_at ISO8601 timestamp The last time the user was updated.
